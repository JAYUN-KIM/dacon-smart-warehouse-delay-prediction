### 2. a150: 과한 tail push 실패

`submission_a150_012.csv`는 public `10.0080170747`로 후퇴했습니다. OOF와 public-guided curve만 보고 tail을 강하게 올리는 방식은 위험했습니다.

이 시점의 판단은 다음과 같습니다.

- p99/max를 올리는 것은 필요하지만 과하면 바로 public에서 악화됩니다.
- tail 강화는 전체 예측에 균일하게 적용하면 안 됩니다.
- 창고 운영 압력이나 미래 slot 위치와 결합해야 합니다.

### 3. a151: queueing/domain reallocation

외부 데이터를 직접 사용하지는 않았지만, queueing theory와 warehouse operation 관점에서 feature를 다시 설계했습니다.

사용한 아이디어는 다음과 같습니다.

- utilization과 capacity gap
- Kingman/VUT 계열 waiting-time proxy
- fork-join bottleneck
- path friction, tech friction
- future queue shock

이 구조로 만든 `submission_a151_886.csv`가 public `10.0068002221`을 기록했습니다.

### 4. a152/a153: 새 축 실험

9점대 진입을 위해 기존 방식을 버리고 더 근본적인 축을 찾았습니다.

- `a152`: train target profile 기반 scenario profile transport
- `a153`: scenario-level mean transfer

둘 다 새 축이었지만, 내부 지표와 후보 분포상 마지막 제출 후보로는 약했습니다. 특히 scenario 평균을 직접 흔드는 것보다, 같은 scenario 안 slot 위치를 조정하는 방식이 더 public에 잘 맞았습니다.

### 5. a154: next_30m phase-lead

대회 정의를 다시 확인했습니다. target은 현재 시점의 지연이 아니라 향후 30분 평균 지연입니다. 따라서 현재 slot 예측에는 앞으로 1~2 slot 뒤의 운영 압력과 지연 신호가 반영되어야 한다고 판단했습니다.

`a154`에서는 현재 best prediction을 같은 scenario 안에서 미래 방향으로 phase-lead했습니다.

- OOF: `8.572104`에서 `8.570084`까지 개선
- 문제: p99/max tail이 낮아져 public에서 힘이 부족할 수 있음
